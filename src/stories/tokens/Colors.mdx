# Colors

The whole site lives in a six-stop grayscale. No color accent. No brand blue.
The restraint **is** the brand. If a visual needs color to communicate, it
probably needs a sharper thought first.

Tokens are defined once in `src/index.css` as HSL triplets on `:root`,
then mapped by Tailwind to semantic names. Everything downstream —
components, charts, motion — reads through the semantic layer.

## Semantic tokens

| Token | Value | Use |
| --- | --- | --- |
| `--background` | ink-150 | Page surface |
| `--foreground` | ink-950 | Body text, primary ink |
| `--card` | ink-200 | Elevated surface |
| `--primary` | ink-950 | Filled buttons, emphasis |
| `--primary-foreground` | ink-150 | Text on primary |
| `--muted-foreground` | ink-500 | Secondary copy |
| `--border` | ink subtle | 1px hairlines, input frames |
| `--destructive` | red | Error, delete — never marketing |

## Rules

- Never use hex. Always reference the semantic token.
- Never hand-roll opacity for ink — use `text-foreground/70` and keep the source of truth in the base color.
- If you find yourself reaching for a new hue, write a memo arguing for it first.
