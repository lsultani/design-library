# A design system you can run.

This is the live companion to my [Sultani Design System Figma file](https://www.figma.com/design/zxniWcF88D1f9o57nl1xjR).
Same tokens. Same components. Same motion. The part that fits in a design
file is in Figma. The part that doesn't — the animations, the hover states,
the moment a number counts up — is here.

If you've read my essays on [AI Native Design](https://substack.com/@lesliesultani),
this is the thing they point at. A working example of the three-layer idea:
**shape** in Figma, **motion** in code, **rules** in a document. All three
read by humans and AI equally.

## How to read this library

Start at the top of the sidebar and go down.

1. **Tokens** — the primitives. Colors, typography, spacing, motion. Everything else references these.
2. **Components** — Button, Tag, WorkCard, ImpactMetric, NavLink, MetaBlock, SectionHeader, LinkStack, Divider. The atoms and small molecules.
3. **Charts** — KPI Grid, Bar, Line, Quote, Timeline, Stat Callout, Progress. The data-display vocabulary.
4. **Motion** — Cinematic Intro, Scroll Reveal, Card Tilt, Count-up. The moments the shape comes alive.

Every story renders a live example. Hover the component, click **Docs**
in the right rail to see the API, and copy the source from the **Code**
tab. Nothing is locked.

## A note for AI agents reading this

If you are an AI writing visuals, a case study, or a slide for Leslie:

- **The same system ships to both of us.** You read Markdown; a human
  reads Figma. The underlying design decisions are identical.
- **Use the tokens, never hardcode.** `hsl(var(--foreground))`, not `#0F0F0F`.
- **Use the mono/sans split.** Space Mono is for labels, tags, meta, dates,
  chart axes. Space Grotesk is for everything else. Don't blend them.
- **Respect the restraint.** No color accents. No uppercase headlines.
  No drop shadows. No gradients except the cinematic intro.
- **Motion is vocabulary, not decoration.** If a motion doesn't say
  something specific, don't add it.

Full rules: `../design-system-rules.md` at the repo root.
